Highlight Detection in Soccer Video

Objective

We propose a novel and computationally efficient framework for soccer video highlight detection. The proposed framework includes some new low-level soccer video processing algorithms, such as shot boundary detection, grass color region detection, view type classification, and play position classification, as well as several higher-level algorithms for slow-motion replay detection, score change detection and goal event detection.The system can output two types of summaries: i) all slow-motion replay segments in a game, ii) all goals in a game.

Overview and Challenges

With the rapid development of multimedia production and transmission technology, how to more effectively, more accurately analyze a large scale of sports video data and extract sports video summarization according to users preferences is becoming a meaningful and challenging topic. Therefore, sports video processing needs to be completed automatically and the processing results must be semantically meaningfully. Besides, how to computationally efficiently and robustly detect highlight event in soccer video is another challenge.

The Journey

This project lasted more than half year of my senior and was chosen as my undergraduation project and bachelor thesis. The whole system and its demo project are designed and developed by myself. Among all the highlight events in soccer video, slow-motion replay and goal events weigh most. In the proposed system, we think some cues in videos are helpful to detect highlight event, so I take soccer video analysis as the first step. In the system, I design some new low-level soccer video processing algorithms, such as shot boundary detection, grass color region detection, view type classification, and play position classification, which use some visual features. Then I introduce new algorithms for fully automatic and computation efficient detection of slow-motion replay and goal event in soccer videos. One two-hour long video costs less than two minutes useing a single cpu in our system. Finally, my efficient and effective framework for soccer video analysis and summarization that combines these algorithms in a scalable fusion outputs two types of summaries: i) all slow-motion replay segments in a game, ii) all goals in a game and are demonstrated in the web demo.

Timeline: December 2012 - September 2013

MSR-Bing Image Retrieval Challenge 2013

Objective

This study addresses approaches for MSR-Bing Image Retrieval Challenge to assess the relevance on a pair of query term and image. Our approaches aim to boost the performance of web scale image retrieval, where images in the initial list (indexed by query term) can be re-ranked based on their relevances. One year BING image retrieval logs are used to develop the relevance assessment system. Several visual features are employed to describe one image. A visual similarity learning algorithm is introduced to train a weighted image similarity.

Overview and Challenges

The task of MSR-Bing Image Retrieval Challenge to assess the relevance between query terms and images. The system need to return a floating point score to reflect their relevance. The single score does not play any significant role. We more concern the score order of images under the same query term. The initial set of images indexed by text can be re-ranked based on the score so that the image search engine can be optimized. Two kinds of framework are presented for the Challenge:)online classification scheme based on semantic information extracted from the click data;2)cluster-based query-image relevance assessment. Furthermore, A novel visual similarity learning algorithm is introduced to train a weighted image similarity.

The Journey

The system consist of two modules. The first module includes two kinds of schemes. We firtly propose a framwork of online classification scheme. Related images are retrieved by indexing the clicked queries in database based on test query. The relevance of the query-image pair is scored by computing the visual similarity between the test image and images indexed by the query. Then we introduce another framwork of cluster-based query-image relevance assessment, in which the score of a query-image pair is given through computing the visual similarity between the test image and database samples which are relevant in the text domain. Some visual features are employed to describe the visual contents. Finally, we propose a new algorithm of visual similarity learning. Instead of using purely visual similarity measurement, we learn a weighted cosine similarity. The challenge result is excellent among all the teams worldwide.

IPv4-based SYN Flood Attack

Objective

SYN Flood attack is currently one of the most popular ways of DoS (Denial of Service attacks), which takes advantage of the TCP protocol loophole with sending a large number of forged TCP connection requests, and making the resources of the attacked servers depleted (CPU full load or memory insufficient).

Overview and Challenges

Denial of Service attack is that an attacker tries different ways to stop the target machines or servers providing services. The basic principle of this method of attack is based on the process of TCP connection. TCP is different from UDP, where the former is based on the connection and TCP connection should be built before transferring datas between server and client. The connection process is called three-way handshake in TCP protocol. We packet and send a large of numbers of forged IP/Tcp packages to a target machine with the open source winpcap lib.

The Journey

This project was designed and conducted in my junior, lasting approximately two months and was selected to participate in BUPT Undergraduate Innovation Project. Fortunately, we gained the class II award among all the teams. In the attack system, we should firstly packet the IP/Tcp packages with the open source winpcap lib, which includes some basic data process, such as populating IP header, TCP header ,Etherner header and checksum function. Then we use a fake IP address to attack a machine with IP address we know in advance. In addition, we should make sure the 8080 port of the target machine is open. The attack effect is rather obvious that the CPU utilization of the target machine ascend suddenly nearly approaching for 100%, which make the server unresponsive. The attack process above called SYN Flood.
